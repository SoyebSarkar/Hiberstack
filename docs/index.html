The Solution

Hiberstack acts as a smart proxy and manager. It tracks collection usage and moves data between active memory and durable storage based on demand.

HOT Active in RAM
DRAINING Snapshotting
COLD On Disk / S3
LOADING Hydrating

Lifecycle States

HOT

Collection is fully loaded in Typesense. All R/W operations are instantaneous.

DRAINING

Writes are buffered/blocked while Hiberstack takes a snapshot and prepares to unload.

COLD

Index is deleted from RAM but exists as a compressed snapshot in durable storage.

LOADING

Triggered by a query. Hiberstack blocks the request, restores the snapshot, then serves the result.

Who is this for?

Multi-tenant SaaS

Perfect for apps with 10,000+ customers where only 5% are active at any given moment.

Research Tools

Large project-based systems where historical data is searched infrequently but must be fast when accessed.

RAM Ceiling Teams

Infrastructure teams facing $5k/mo+ RAM bills who want to optimize for cost without losing speed.

Note: If you have a single 100GB collection that is always active, Hiberstack won't help you. You need sharding.

Getting Started

# docker-compose.yml
services:
  typesense:
    image: typesense/typesense:26.0
    
  hiberstack:
    image: hiberstack/hiberstack:latest
    environment:
      - TYPESENSE_API_KEY=xyz
      - STORAGE_BACKEND=s3
      - TTL_INACTIVE=3600
    ports:
      - "8108:8108" # Route your traffic here
